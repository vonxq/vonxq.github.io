相关问题

  1. Form 是不是高阶组件
    我的理解中应该不是，高阶组件是个纯函数且返回一个接受组件作为参数的函数，这么定义的话 Form.create()这个函数是一个高阶组件
  2. Form.create()(Component)和 Component 的关系?
    父子关系，拥有互不干扰的生命周期。这个问题理解了问题 3 就不存在了
  3. Form 组件如何解决两个组件生命周期重叠的问题?
    答： 不存在重叠，Form 和 Form.create()(Component)返回的被包裹组件是两个东西
    Form（返回的是个定义了 props 和 className 的原生 form）是定义在被包裹组件中的

高阶组件

Higher Order Component，高阶组件
高阶组件就是一个 React 组件包裹着另外一个 React 组件
组件: 可以是纯函数

官方定义:
高阶组件是一个函数，能够接受一个组件并返回一个新的组件。
组件是将 props 转化成 UI，然而高阶组件将一个组价转化成另外一个组件。
高阶组件是纯函数，没有副作用。

你可以想象在一个大型项目中，订阅 DataSource 并调用 setState 的函数(原文举例，大概就是相同逻辑可能在不同类型组件上都会遇到)将会一次次出现。我们需要将其抽象出来，使得我们能够在一个地方定义逻辑并且在我们的组件中共享。这就是高阶组件的优点。

两个作用:

  1. 属性代理 Props Proxy： HOC 对传给 WrappedComponent W 的 porps 进行操作
  2. 反向继承 Inheritance Inversion： HOC 继承 WrappedComponent W。

通俗点讲，属性代理可以对组件的 props 做一些预处理，再下传 porps 给被包裹组件
