React技术栈简介

Posted on 2017-08-16 Edited on 2022-05-30 In Frontend , React

项目所用到的 React 技术栈如下:

  1. 语法: React 语法、ES6 语法、Less 语法
  2. 相关 lib: React、Dva、react-router、redux、redux-saga、Mock.js、ant-design
  3. 工具：Webpack、webpack-dev-server、roadhog、Bable、dva-cli

语法

React 语法： jsx 语法，就是在 js 中写 html 代码
ES6 语法: 即 ES2015，增加一些新特性，如箭头函数、class 继承(语法糖)、Generator 函数等
Less 语法: 用于 css 的编写，LESS 将 CSS 赋予了动态语言的特性，如变量、继承、运算、函数。LESS 既可以在客户端上运行(支持 IE 6+, Webkit, Firefox)，也可以借助 Node.js 或者 Rhino 在服务端运行，点击查看less 语法简介

相关 lib

React

官方对 React 的定义为用于构建用户界面的 js 库。利用官方提供的组件生命周期钩子函数设计不同功能不同外观的可复用组件，界面由数据驱动，不用自己再重写了。
react 主要特点：

  1. 组件化
  2. 高效 DOM 操作，Virtual DOM 机制，Diff 算法
  3. 单项数据流(Model->View)
  4. 组件生命周期
  5. 声明式 UI 设计 声明式编程和命令式编程的比较

react-router

React 只解决了数据与界面的关联问题，对于一个单页面程序，路由必不可少，react-router 主要用于界面路由，页面的跳转
分为 HashRouter、BrowserRouter

redux

React 的组件间通信规则基本如下:
父到子: 通过传给子组件的 props 传递数据
子到父: 通过父组件提供的回调函数传递数据
兄弟组件: 用父组件作为媒介通信
可以看出，当一个界面较为复杂，组件间嵌套层级深时，组件间数据传递非常困难，所以官方提供了 redux 库来进行大项目中的数据分发。基本原理与 flux 相同

redux-saga

redux 的异步解决方案

Dva

基于 redux、redux-saga 和 react-router 的轻量级前端框架。

Mock.js

用于数据的 mock，伪造一些数据供前端测试，下面示例造了一个含 20 条数据的数组

ant-design

蚂蚁金服提供的 React 组件库，包含了大部分日常前端开发的常见组件，设计简洁，开箱即用

工具

Bable

浏览器对 ES6 的支持不全，且不能识别 React 语法写得程序，造成写好的程序在浏览器无法成功运行。
为了让我们提前用上浏览器的新特性切不用考虑浏览器支持的问题，babel 就诞生了。
babel 使用相关的 preset 插件对我们的文件进行预处理，使之能够被浏览器识别

babel-cli 和 babel-core:
如果你想要在 CLI(终端或 REPL)使用 babel 就下载 babel-cli，如果想要在 node 中使用就下载 babel-core。
最常用的 babel preset 插件:
babel-preset-react: 预处理 react 语法
babel-preset-es2015: 预处理 ES6 语法
babel-preset-stage-0： 预处理 ES7 语法

Webpack

webpack 是一个前端模块化构建工具，它能把整个工程通过用户配置的 module resolver 进行预处理并输出。
官方介绍如下:

webpack 是一个现代 JavaScript 应用程序的模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成少量的 bundle - 通常只有一个，由浏览器加载。
它是高度可配置的，但是，在开始前你需要先理解四个核心概念：入口(entry)、输出(output)、loader、插件(plugins)。
一个简单的 react+babel+webpack 的示例:
react-hello-world

webpack-dev-server

webpack-dev-server 是一个小型的 Node.js Express 服务器,它使用 webpack-dev-middleware 来服务于 webpack 的包,除此自外，它还有一个通过 Sock.js 来连接到服务器的微型运行时。
详解 webpack-dev-server 的使用

dva-cli

dva 官方提供的一个命令行工具，用于快速构建一个 dva 程序，使用户不用为项目环境的搭建费心。

roadhog

用 json 格式进行配置的 cli 工具,提供了 mock 功能，可配置 webpack(有些接口未开放)
官方介绍如下:

roadhog 是一个 cli 工具，提供 server、 build 和 test 三个命令，分别用于本地调试和构建，并且提供了特别易用的 mock 功能。命令行体验和 create-react-app 一致，配置略有不同，比如默认开启 css modules，然后还提供了 JSON 格式的配置方式。
本地开发:
$ roadhog server
打包发布:
$ roadhog build
测试，默认会跑 ./test 目录下的所有文件:
$ roadhog test
