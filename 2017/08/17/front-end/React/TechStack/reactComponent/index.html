React组件及其生命周期(React核心概念介绍)

Posted on 2017-08-17 Edited on 2022-05-30 In Frontend , React

单说 React 的话，组件的构建与使用基本就是 React 的全部内容了，所以此文也可以叫 React 使用介绍。

React 五大核心概念

包含 JSX、组件、组件的 props&state、组件 API、组件类型，可以看出 React 的核心都在组件化上了

JSX

React 是通过 JSX 语法扩展来实现的

基本 JSX 语法

  1. Component 可以嵌套
  2. className 代替 class
  3. JavaScript 表达式需要用 {} 括起来，会执行并返回结果。
  4. Mapping Arrays to JSX,可以把数组映射为 JSX 元素列表
  5. Spread Attribute：这是 JSX 从 ECMAScript6 借鉴过来的很有用的特性，用于扩充组件 props 。

组件

React 组件能够像原生的 HTML 标签一样输出特定的界面元素，并且也能包括一些元素相关逻辑功能的代码。
组件是一个由具有特定外观特定功能与数据集合的前端页面组成单位。

组件的 Props&state（属性与状态）

props 是一个组件暴露给使用者(父组件)的一些属性，父组件可通过 props 的配置对组件进行一些初始化，如对一个 Table 传入 dataSource 进行展示
State 是组件内部状态，组件内部自己维护状态变化。外部使用者(父组件)不会关心这些状态
注意:
props 和 state 的变化都会引起组件的重绘(render)

组件 API

一些方法。常用如下：

  1. render、setState、constructor
  1. render 中返回将要被渲染的组件内容，纯函数组件不用 render，直接 return 组件内容即可
  2. setState 用于更新组件状态，会引起组件重绘(render 被执行)
  3. constructor 为当用 ES6 方式构建组件时的构造函数，主要进行 state 的初始化操作
  1. 生命周期函数，如图所示
    常用生命周期函数:
    1. componentDidMount: 组件刚 mount 上后执行，可在此函数中进行一些组件的初始化操作 2. componentWillReceiveProps: 组件接收到新的 props 后执行，参数为 nextProps,可在此进行一些组件更新操作

组件类型

组件分类：

  1. 容器组件：关注数据交互，如加载服务器端数据
  2. 展示组件：关注 UI 逻辑，用来展示或隐藏内容
    高阶组件（higher-order components，HOCs）：可以把它理解为一个工厂方法，你可以传入一个组件并得到一个 HOC 返回的附加了更多功能的新组件。HOC 不能直接在 render 方法中调用。

组件定义与使用

定义组件方式：

  1. React.createClass 最早，兼容最好
  2. ES6 Class
  3. 函数式组件（无状态组件，没 state）

纯函数方式定义组件

特点

与函数创建方式相同，props 作为函数参数，return 组件布局,每次调用从头执行到尾
使用 Functional Component 的写法，单纯渲染 UI
A. 组件不能被实例化，节约内存，提升一定的性能
B. 组件不能访问 this 对象
C. 组件无法访问生命周期方法
D. 只能访问 props
E. 无状态组件内部其实是可以使用 ref 功能的，虽然不能通过 this.refs 访问到，但是可以通过将 ref 内容保存到无状态组件内部的一个本地变量中获取到。

构建出的组件特点

React.createClass 定义的组件

  1. This 自绑定，自动获取当前组件实例对象
  2. React.createClass 在创建组件时，有关组件 props 的属性类型及组件默认的属性会作为组件实例的属性来配置，其中 defaultProps 是使用 getDefaultProps 的方法来获取默认组件属性的
  3. React.createClass 创建的组件，其状态 state 是通过 getInitialState 方法来配置组件相关的状态

组件使用相关 tips

  1. 组件使用时首字母必须大写，否则无法识别
  2. 有状态组件才能用生命周期钩子函数
  3. 组件更新时机：props/state 的变化都会触发 render()更新界面，同时也可以在生命周期函数中对界面进行处理
  4. shouldComponentUpdate 默认返回 true，重写可避免不必要的更新

组件设计

尽量无状态，交由父组件维护自身状态并通过 props 传值
状态 state： 状态需要包含那些仅与自身有关(不需要父组件决定)并且在组件的回调函数中会发生变化（用户行为改变状态），并且会体现在 UI 上的信息。

组件划分思想

单一职责原则
好的组件特性:
（1）可组合（Composeable）：一个组件易于和其它组件一起使用，或者嵌套在另一个组件内部。如果一个组件内部创建了另一个组件，那么说父组件拥有（own）它创建的子组件，通过这个特性，一个复杂的 UI 可以拆分成多个简单的 UI 组件；
（2）可重用（Reusable）：每个组件都是具有独立功能的，它可以被使用在多个 UI 场景；
（3）可维护（Maintainable）：每个小的组件仅仅包含自身的逻辑，更容易被理解和维护；
（4）可测试（Testable）：因为每个组件都是独立的，那么对于各个组件分别测试显然要比对于整个 UI 进行测试容易的多。

建议大型可复用组件的开发步骤

  1. 先按具体做出一个 Demo 组件，组件内的一些变化的值(将来要作为 props 提供的)用一个对象提供
  2. 对象定义好后 const 取出对象内部属性值，在 return 的内容中使用
  3. 及时修改不合理的 props,若组件过于复杂需进一步拆分组件
  4. 在父组件中使用此组件，提供对象相同的一些 props 属性
  5. 将 const 来源替换成 props，并为组件定义 propTypes
